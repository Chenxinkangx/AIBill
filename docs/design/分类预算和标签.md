# 分类预算与标签设计说明

## 1. 设计背景

在记账系统中，一笔消费往往不只具有一种含义。

例如：

* 和女朋友吃火锅，既是一次“吃饭”，也是一次“约会消费”；
* 晚饭买了薯片和可乐，既是“吃的”，也是“零食消费”；
* 和朋友喝奶茶，既是“社交”，也是“饮品消费”。

如果系统只允许一笔消费属于一个普通分类，就会出现统计不准确的问题。

因此，AIBill 将消费记录拆分为两个概念：

* **分类预算**
* **标签**

二者分别承担不同职责。

---

## 2. 分类预算是什么？

分类预算用于表示：

> 这笔钱应该从哪个预算中扣除。

它的核心作用是控制预算剩余金额。

例如用户设置了以下预算：

| 分类预算 |  每月预算 |
| ---- | ----: |
| 餐饮   | 600 元 |
| 零食   | 150 元 |
| 约会社交 | 500 元 |
| 交通   | 200 元 |
| 学习   | 300 元 |

当用户记录一笔消费时，系统需要明确这笔消费应该扣除哪一个预算。

例如：

```text
和女朋友吃火锅，花费 168 元
```

这笔钱虽然本质上是吃饭，但用户可能认为它主要属于约会支出，因此可以记录为：

```text
分类预算：约会社交
金额：168 元
```

这样系统会从“约会社交预算”中扣除 168 元，而不是从“餐饮预算”中扣除。

---

## 3. 分类预算的特点

分类预算具有以下特点：

### 3.1 一笔消费只能属于一个分类预算

为了保证预算剩余金额准确，一笔消费只能扣除一个预算。

例如：

```text
分类预算：约会社交
金额：168 元
```

这笔钱只会扣除“约会社交预算”，不会同时扣除“餐饮预算”。

这样可以避免重复扣款的问题。

如果一笔 168 元的消费同时扣除“餐饮预算”和“约会社交预算”，用户看到的预算消耗会变成两边都减少 168 元，容易造成误解。

---

### 3.2 分类预算用于预算控制

分类预算主要用于回答：

```text
这个月每个预算还剩多少钱？
```

例如：

```text
餐饮预算：600 元，已用 57 元，剩余 543 元
零食预算：150 元，已用 18 元，剩余 132 元
约会社交预算：500 元，已用 188 元，剩余 312 元
```

因此，分类预算是系统中最核心的预算统计依据。

---

## 4. 标签是什么？

标签用于描述一笔消费的附加属性。

它的核心作用不是扣预算，而是帮助用户从不同角度分析消费。

例如：

```text
和女朋友吃火锅，花费 168 元
```

可以记录为：

```text
分类预算：约会社交
标签：约会、餐饮、火锅、晚餐
```

这里的含义是：

* 从预算角度看，这笔钱属于“约会社交”；
* 从消费属性看，它也是“餐饮”“火锅”“晚餐”。

---

## 5. 标签的特点

### 5.1 一笔消费可以有多个标签

因为一笔消费可能有多个属性，所以标签可以有多个。

例如：

```text
买薯片和可乐，花费 18 元
```

可以记录为：

```text
分类预算：零食
标签：零食、吃的、冲动消费、晚饭替代
```

这表示：

* 预算上，它扣除“零食预算”；
* 分析上，它属于“吃的”“冲动消费”“晚饭替代”。

---

### 5.2 标签不直接扣预算

标签只用于分析、筛选和复盘，不直接影响预算剩余。

例如：

```text
分类预算：约会社交
标签：餐饮、火锅、约会
金额：168 元
```

这笔钱只会扣除“约会社交预算”。

虽然它带有“餐饮”标签，但不会扣除“餐饮预算”。

---

## 6. 分类预算和标签的区别

| 对比项      | 分类预算          | 标签               |
| -------- | ------------- | ---------------- |
| 核心作用     | 决定扣哪个预算       | 描述消费属性           |
| 是否影响预算剩余 | 影响            | 不直接影响            |
| 一笔消费可选数量 | 只能一个          | 可以多个             |
| 主要用于     | 预算控制、剩余金额统计   | 筛选、分析、复盘         |
| 示例       | 餐饮、零食、约会社交、交通 | 餐饮、火锅、约会、外卖、冲动消费 |

简单来说：

```text
分类预算 = 这笔钱从哪个钱包扣
标签 = 这笔钱还有哪些属性
```

---

## 7. 实际例子

假设用户本月设置了以下预算：

```text
餐饮预算：600 元
零食预算：150 元
约会社交预算：500 元
```

本月记录了以下消费：

| 消费内容    |    金额 | 分类预算 | 标签          |
| ------- | ----: | ---- | ----------- |
| 公司楼下吃盖饭 |  22 元 | 餐饮   | 餐饮、午餐、工作日   |
| 买薯片和可乐  |  18 元 | 零食   | 零食、吃的、冲动消费  |
| 和女朋友吃火锅 | 168 元 | 约会社交 | 约会、餐饮、火锅、晚餐 |
| 点外卖晚饭   |  35 元 | 餐饮   | 餐饮、外卖、晚餐    |
| 和朋友喝奶茶  |  20 元 | 约会社交 | 社交、奶茶、吃的    |

---

## 8. 按分类预算统计

预算页面只根据“分类预算”统计。

结果如下：

```text
餐饮预算：
600 - 22 - 35 = 543 元

零食预算：
150 - 18 = 132 元

约会社交预算：
500 - 168 - 20 = 312 元
```

这个统计结果用于告诉用户：

```text
每个预算还剩多少钱。
```

---

## 9. 按标签分析

如果用户想知道：

```text
我这个月所有和吃有关的钱花了多少？
```

这时只看“餐饮预算”是不够的。

因为有些吃的消费并没有扣“餐饮预算”：

* 火锅扣的是“约会社交预算”；
* 薯片可乐扣的是“零食预算”；
* 奶茶扣的是“约会社交预算”。

因此，系统可以通过标签进行分析。

所有和“吃”相关的消费包括：

```text
公司楼下吃盖饭：22 元
买薯片和可乐：18 元
和女朋友吃火锅：168 元
点外卖晚饭：35 元
和朋友喝奶茶：20 元
```

合计：

```text
22 + 18 + 168 + 35 + 20 = 263 元
```

也就是说：

```text
虽然餐饮预算只花了 57 元，
但本月所有和吃有关的消费一共是 263 元。
```

这就是标签的作用。

---

## 10. 推荐规则

AIBill 可以采用以下规则：

### 10.1 预算扣减规则

```text
每笔消费只能选择一个分类预算。
```

这样可以保证预算剩余金额准确，不会重复扣除。

---

### 10.2 标签记录规则

```text
每笔消费可以添加多个标签。
```

标签用于补充说明这笔消费的属性，方便后续筛选、统计和 AI 分析。

---

### 10.3 页面统计规则

```text
预算页面：按分类预算统计。
分析页面：按标签统计。
```

也就是说：

* 用户想看“还剩多少钱”，看分类预算；
* 用户想看“钱为什么花掉了”，看标签分析。

## 12. 总结

分类预算和标签的区别可以总结为：

```text
分类预算负责预算扣减，标签负责消费分析。
```

或者更直白地说：

```text
分类预算回答：这笔钱从哪个预算里扣？
标签回答：这笔钱还有哪些属性？
```

采用这种设计后，AIBill 可以同时做到：

1. 预算剩余金额准确；
2. 消费记录不重复扣款；
3. 后续可以从多个角度分析消费；
4. AI 自动记账时可以更灵活地识别消费场景。
