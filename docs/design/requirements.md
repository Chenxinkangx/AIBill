# AI 预算记账助手 — 需求文档

> 核心定位：不是传统记账软件，而是**以预算控制为核心的 AI 记账工具**。
> 核心回答：**"我这个月还能花多少钱？"**

---

## 1. 项目概述

| 项目 | 内容 |
|------|------|
| **项目名称** | AI 预算记账助手（AIBill） |
| **目标用户** | 个人，每月有固定预算，希望控制开支 |
| **产品定位** | 预算管理，而不是流水记账 |
| **产品 slogan** | 这个月还能花多少钱？ |

---

## 2. 产品目标与核心原则

### 2.1 核心原则

- **预算先行**：用户先设预算，再记账，系统始终围绕预算状态做反馈。
- **AI 辅助，人工确认**：AI 负责快速录入，用户负责确认，AI 不自动写入数据。
- **账单是原始数据，预算是计算结果**：不存储"剩余预算"字段，所有预算状态由账单动态计算得出。
- **本地优先**：V1 不依赖云端，数据完全在本地，用户拥有完整所有权。

### 2.2 预算与分类的关系

- **月总预算** = 用户本月计划可支配的总金额上限。
- **分类预算之和与月总预算的关系**：分类预算之和可以小于、等于或大于月总预算，系统需要显示差额。

| 情景 | 示例 |
|------|------|
| 分类预算 < 总预算 | 总预算 3700，分类合计 3600，显示**未分配预算：100** |
| 分类预算 = 总预算 | 总预算 3700，分类合计 3700，显示**预算已全部分配** |
| 分类预算 > 总预算 | 总预算 3700，分类合计 4000，显示**已超出总预算：300** |

### 2.3 收入的处理方式（V1）

- V1 **同时支持支出和收入两种记录类型**。
- **支出参与预算计算**：支出账单会扣减对应分类和总预算。
- **收入仅用于统计展示**，不参与预算计算，不影响"本月剩余"和"今日建议可花"。
- AI 记账可同时识别支出和收入；手动记账也支持两类。
- **"收入"是一个系统内置的特殊分类**，存在 Category 表中，标记 `budgetable = false`，不参与预算设置和计算。
- 收入分类前端使用固定标识，不在普通分类列表中展示给用户，不允许删除。

## 3. 核心使用流程

### 3.1 AI 记账主流程

```
用户输入自然语言
     ↓
AI 解析（金额、分类、日期、类型）
     ↓
展示待确认账单列表（ParsedRecordItem）
     ↓
用户操作：确认 ✓ / 修改分类 / 删除条目
     ↓
用户点击"确认保存"
     ↓
批量转换为 RecordItem 写入 IndexedDB
     ↓
预算动态重新计算（首页即时更新）
     ↓
提示保存成功
```

### 3.2 手动记账兜底流程

```
用户打开手动记账表单
     ↓
填写：金额、分类、类型（支出/收入）、日期、备注
     ↓
点击保存
     ↓
写入 IndexedDB
     ↓
预算动态重新计算
```

### 3.3 首次启动流程

```
首次打开 App
     ↓
自动创建预置分类（8 个默认分类）
     ↓
引导用户设置本月总预算和分类预算
     ↓
进入首页 Dashboard
```

> 如果用户未设置预算，首页显示空状态，并引导进入预算页设置。

## 5. V1 功能需求详情

### 5.1 首页 Dashboard

- 展示**本月剩余金额**（核心数据，字体最大）
- 展示**今日建议可花金额**
- 展示**本月已花金额**和**月总预算**
- 展示各分类预算进度条（名称、已花、预算、使用比例、剩余）
- 展示**未分配预算**或**超分配提示**
- 月份选择器（默认当前月份，可切换查看历史）
- 快速记账入口按钮

**空状态**：
- **未设置本月预算**：显示引导文案"请先设置本月预算"，按钮跳转预算页。
- **切换到未设置预算的月份**：显示空状态"该月份还未设置预算"，提供两个按钮：
  - "手动设置预算"：跳转预算页，自动切换到该月份
  - "使用默认预算创建"：以上月预算或全局默认预算填充该月
- **月份有预算但无账单**：正常显示预算数据，各分类进度为 0%。

**月份切换显示规则**：
- 当前月份：显示"今日建议可花"
- 历史月份：不显示"今日建议可花"，改为显示"当月结余"

### 5.2 记账

**AI 记账**：
- 自然语言输入框 + "智能识别"按钮
- 支持同时识别支出和收入（如"工资到账 6000，午饭 18"）
- 识别结果以卡片列表展示，每条可修改分类、删除
- 置信度 < 0.7 的条目标记"请确认"
- 点击"确认保存"批量入库
- AI 解析失败时提示"暂时无法识别，请尝试简化描述或使用手动记账"

**手动记账**：
- 金额（必填，必须 > 0）
- 分类（必填，下拉选择已有分类 / 收入类）
- 类型（支出 / 收入）
- 日期（默认当天，可修改）
- 备注（可选）

### 5.3 预算

- 设置月度总预算（按月存储，支持切换月份查看历史）
- 为每个分类设置预算金额
- 显示分类预算合计与总预算的差额
- 允许分类预算之和大于总预算（显示超分配警示）
- 允许分类预算之和小于总预算（显示未分配金额）
- 月份选择器切换

### 5.4 账单列表

- 默认展示当前月份，支持月份切换
- 按日期分组展示（降序）
- 每条显示：标题、金额、分类、类型标识
- 点击展开操作：编辑 / 删除
- 按分类下拉筛选

### 5.5 设置

- **导出 JSON**：导出全部数据为 JSON 文件下载
- **导入 JSON**：选择文件，全量覆盖当前数据（导入前提示先备份）
- **清空数据**：清除所有本地数据，二次确认后执行
- **AI API Key 配置**：输入 AI 接口的 API Key，保存在 localStorage
- **货币单位**：切换 CNY / 其他
- **关于项目**：版本号、说明

### 5.6 数据管理

- 使用 Dexie.js 封装 IndexedDB 存储所有主体数据
- localStorage 仅用于保存轻量设置（API Key、货币单位等）
- 支持导出全部数据为 JSON 文件
- 支持导入 JSON 文件**全量覆盖恢复**
- 导入前校验文件格式，失败时提示"文件格式错误，数据未变更"
- 提供清空数据入口（二次确认）

#### 5.6.1 JSON 导出格式

导出文件必须包含版本信息，便于后续迁移兼容：

```json
{
  "app": "AIBill",
  "version": "1.0.0",
  "exportedAt": "2026-06-09T10:00:00.000Z",
  "data": {
    "categories": [],
    "monthlyBudgets": [],
    "categoryBudgets": [],
    "records": [],
    "settings": {}
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| app | string | 固定为 "AIBill" |
| version | string | 导出时的版本号，用于导入时兼容判断 |
| exportedAt | string | ISO 格式导出时间戳 |
| data | object | 核心数据容器，包含所有表数据 |

## 7. 数据结构

### 7.1 Category（分类）

```typescript
interface Category {
  id: string;
  name: string;          // 餐饮、娱乐、学习、收入...
  icon?: string;         // 可选的图标
  color?: string;        // 对应颜色
  order: number;         // 排序序号
  archived?: boolean;    // 是否已归档（停用）
  system?: boolean;      // 是否系统内置（如"收入"），不允许删除和编辑
  budgetable: boolean;   // 是否参与预算设置和计算，收入分类为 false
}
```

> V1 **不允许删除已有账单引用中的分类**。如果分类下有账单，只允许归档（`archived = true`），归档后不显示在新账单的分类选择中。
>
> **系统内置收入分类**：`{ id: "income", name: "收入", system: true, budgetable: false, order: -1 }`。存入普通 Category 表，但前端不展示在分类列表中，不参与预算设置和计算。

**预置分类（V1）**：以下 8 个分类均参与预算设置和计算。"收入"为系统内置特殊分类，不在此表内，不参与预算。

| 分类 | 建议预算（参考） |
|------|------------------|
| 餐饮 | 1500 |
| 娱乐 | 400 |
| 学习 | 300 |
| 交通 | 200 |
| 健身 | 200 |
| 日用品 | 300 |
| 恋爱/社交 | 500 |
| 其他 | 300 |

### 7.2 MonthlyBudget（月度总预算）

```typescript
interface MonthlyBudget {
  id: string;
  month: string;           // "2026-06"
  totalBudget: number;     // 月总预算
  monthlyIncome?: number;  // 月收入（可选，仅统计展示）
  createdAt: string;       // ISO 时间戳
  updatedAt: string;       // ISO 时间戳
}
```

> 按月存储总预算，支持查看历史月份数据。
> **唯一约束**：`month` 唯一，同一月份只能存在一条月度总预算记录。

### 7.3 CategoryBudget（分类预算）

```typescript
interface CategoryBudget {
  id: string;
  categoryId: string;
  month: string;           // "2026-06"
  amount: number;          // 该分类本月预算金额
  createdAt: string;       // ISO 时间戳
  updatedAt: string;       // ISO 时间戳
}
```

> 预算状态不单独存储，由账单数据动态计算得出。
> **唯一约束**：`month + categoryId` 联合唯一，同一月份同一分类只能存在一条预算记录。

### 7.4 RecordItem（账单记录）

```typescript
interface RecordItem {
  id: string;
  title: string;
  amount: number;          // 必须 > 0
  categoryId: string;      // 支出关联普通分类；收入统一使用系统固定 categoryId = "income"
  type: 'expense' | 'income';
  date: string;            // "2026-06-09"
  note?: string;
  source: 'manual' | 'ai';
  createdAt: string;       // ISO 时间戳
  updatedAt: string;       // ISO 时间戳，编辑时更新
}
```

### 7.5 ParsedRecordItem（AI 解析结果）

```typescript
interface ParsedRecordItem {
  title: string;
  amount: number;               // 必须 > 0
  categoryId?: string;
  categoryName: string;         // AI 识别的分类名称
  type: 'expense' | 'income';
  date: string;                 // "2026-06-09"
  confidence: number;           // 置信度 0-1
  rawText?: string;             // 原始输入文本片段
}
```

> 流程：`ParsedRecordItem` → 用户确认 → 转换为 `RecordItem` 入库

## 8. 核心计算规则

> **重要原则**：所有预算状态由账单数据动态计算得出，不存储"剩余预算"字段。

### 8.1 本月已花金额

```
本月已花 = 当前月份所有 type = 'expense' 的账单金额之和
```

### 8.2 分类已花金额

```
分类已花 = 当前月份该分类下所有 type = 'expense' 的账单金额之和
```

### 8.3 本月剩余金额

```
本月剩余 = 月总预算 - 本月已花

如果本月剩余 < 0 → 显示 0，并提示"本月已超支"
```

### 8.4 分类剩余金额

```
分类剩余 = 分类预算 - 分类已花
```

### 8.5 今日建议可花金额

```
今日建议可花 = 本月剩余金额 / 包含今天在内的本月剩余天数
```

**示例**：
- 6 月 9 日，6 月共 30 天
- 剩余天数 = 30 - 9 + 1 = 22 天（**包含今天**）
- 本月剩余 2,360 → 今日建议可花 = 2,360 ÷ 22 ≈ 107 元

> 如果本月剩余 < 0，今日建议可花显示为 **0**。

> **历史月份显示规则**：查看当前月份时显示"今日建议可花"；查看历史月份时，改为显示**"当月结余"**（即该月剩余金额），不显示"今日建议可花"。

### 8.6 月份进度

```
月份进度 = 当前日期 / 当前月份总天数（保留两位小数）
```

**示例**：6 月 9 日 → 月份进度 = 9 / 30 = 30%

### 8.7 分类预算合计

```
分类预算合计 = 所有分类预算金额之和
```

### 8.8 未分配 / 超分配

```
差额 = 月总预算 - 分类预算合计

差额 > 0 → 未分配预算：{差额}
差额 = 0 → 预算已全部分配
差额 < 0 → 已超出总预算：{差额的绝对值}
```

### 8.9 预算状态颜色规则

| 状态 | 颜色 | 条件 |
|------|------|------|
| 已超预算 | 红色 | 使用率 >= 100% |
| 预警 | 黄色/橙色 | 使用率 > 月份进度 + 20%，且 < 100% |
| 正常 | 绿色 | 其他情况 |

**示例**：
- 6 月 9 日，月份进度 30%
- 餐饮使用率 45% → 45% < 30% + 20%（50%），绿色
- 餐饮使用率 55% → 55% > 50%，黄色预警
- 餐饮使用率 100% → 红色超支

## 9. AI 记账交互规范

### 9.1 触发方式

用户在输入框中输入自然语言，点击"智能识别"按钮触发 AI 解析。

### 9.2 解析规则

- 识别每条消费记录的**标题、金额、分类、日期、类型（支出/收入）**
- 金额支持各种写法："18"、"18元"、"¥18"、"18块钱"
- 分类自动匹配已有分类名称（见 9.3 分类映射）
- 日期识别规则见 9.4

### 9.3 分类关键词映射

以下关键词映射用于 AI Prompt，帮助 AI 准确匹配分类：

| 分类 | 关键词 |
|------|--------|
| 餐饮 | 早餐、午饭、晚饭、外卖、食堂、奶茶、咖啡、零食、水果、饮料、面包 |
| 交通 | 地铁、公交、打车、滴滴、共享单车、高铁、机票、加油、停车 |
| 学习 | 书、课程、会员、教程、考试、资料、付费知识、网课、文具 |
| 娱乐 | 电影、游戏、聚会、KTV、视频会员、音乐、旅游、景点、桌游 |
| 健身 | 健身房、游泳、运动装备、私教、瑜伽、羽毛球、球拍 |
| 日用品 | 纸巾、洗发水、牙膏、洗衣液、生活用品、收纳、家居 |
| 恋爱/社交 | 约会、礼物、请客、聚餐、红包、鲜花 |
| 收入 | 工资、兼职、退款、红包收入、理财、奖金、补贴 |
| 其他 | 无法判断的消费均归入此项 |

### 9.4 日期识别规则（V1）

| 写法 | 解析结果 |
|------|----------|
| 无日期 | 默认为当天 |
| 今天 | 当天 |
| 昨天 | 当天 - 1 天 |
| 前天 | 当天 - 2 天 |
| 6月9日 | 当年 6 月 9 日 |
| 2026-06-09 | 指定日期 |

> **注意**：复杂相对日期（如上周五、上个月、上个月月底）在 V1 中不强制支持，AI 识别失败时默认为当天，并提示用户确认。

### 9.5 确认流程

```
用户输入 → AI 解析 → 展示 ParsedRecordItem 列表
                          ↓
              用户操作：确认 / 修改分类 / 删除条目
                          ↓
                      用户点击"确认保存"
                          ↓
               转换为 RecordItem 写入 IndexedDB
                          ↓
                      首页预算状态自动更新
```

### 9.6 安全兜底

- AI 解析结果中 `confidence < 0.7` 的条目，在确认页标记为**"请确认"**
- 无法映射到已有分类的条目归入**"其他"**
- 用户**必须**手动点击确认才能保存，AI 不自动写入数据
- AI 解析失败时提示："暂时无法识别，请尝试简化描述或使用手动记账"

### 9.7 空输入处理

- 如果用户输入中**没有明确包含金额的消费/收入记录**（如"今天没花钱""帮我看看预算"），AI 返回**空数组 `[]`**
- 前端收到空数组时提示："未识别到有效账单，请补充金额或使用手动记账"
- 不清空输入框，允许用户修改后重试

## 11. 异常情况处理

### 11.1 AI 解析失败

- 提示用户："暂时无法识别，请尝试简化描述或使用手动记账"
- 不清空输入框，允许用户修改后重试

### 11.2 金额不合法

- 金额为空、小于等于 0 或非数字时，禁止保存
- 在输入框下方显示错误提示
- 手动记账和 AI 确认环节均需校验

### 11.3 分类被删除/归档

- V1 不允许删除已有账单引用中的分类
- 如果分类下有账单，只允许归档（`archived = true`）
- 归档分类不在新增账单时显示，历史账单正常显示

### 11.4 数据导入

- 选择文件后校验 JSON 格式
- 格式错误时提示："文件格式错误，数据未变更"
- 导入采用**全量覆盖模式**：清空当前数据后写入导入文件中的数据
- 导入前提示用户："导入会覆盖当前本地数据，建议先导出备份"

### 11.5 本地数据损坏

- 应用启动时检测数据完整性
- 数据损坏时提示用户，并提供两个入口：
  - **清空数据**：清除所有本地数据，重新开始
  - **导入备份**：从之前导出的 JSON 文件恢复

| 层面 | 建议方案 |

| **AI 接口** | Claude / OpenAI 等 LLM API（自然语言解析） |

| **部署** | V1 单页应用，静态托管，无需后端 |

### 13.1 AI API Key 安全说明（重要）

- V1 为**个人自用版本**，AI API Key 由用户在本机设置，保存在 localStorage 中。
- **该方式仅适合个人本地使用**，不适合公开部署。
- 如果后续公开发布，必须增加后端代理服务，避免 API Key 暴露到前端。
- V1 不考虑 Token 计费展示、Key 过期管理等复杂功能。
